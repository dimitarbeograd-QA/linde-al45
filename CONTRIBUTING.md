# Contributing

## Как да допринесеш
1. Направи промените директно в `index.html` (или създай branch за по-големи промени).
2. Тествай ръчно по чеклиста в [`qa-docs/TEST_CASES.md`](qa-docs/TEST_CASES.md), особено zoom/pan и roles-свързаните кейсове.
3. Commit съобщенията да са кратки и да описват **какво** и **защо** се променя.
4. При промяна в диаграмата/схемите — увери се, че съществуващите номера на части не се разместват неволно.

## Branch/commit конвенция
- `main` е активният branch — директни commit-и са ок за малки проекти от този мащаб.
- За по-рискови промени (напр. admin логика), обмисли отделен branch + преглед преди merge.

## Docs
При добавяне на нова функционалност, обнови съответно:
- [`README.md`](README.md) — ако се добавя нова основна функция
- [`qa-docs/TEST_CASES.md`](qa-docs/TEST_CASES.md) — нов тест кейс
- [`CHANGELOG.md`](CHANGELOG.md) — запис за промяната
