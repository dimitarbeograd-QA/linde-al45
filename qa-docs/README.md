# QA документация — Linde AL45

Тази папка съдържа QA процеса за проекта.

## Съдържание
- [`TEST_PLAN.md`](TEST_PLAN.md) — стратегия, обхват и критерии за приемане.
- [`TEST_CASES.md`](TEST_CASES.md) — детайлни тест кейсове по функционалност.
- Bug report шаблон: [`.github/ISSUE_TEMPLATE/bug_report.md`](../.github/ISSUE_TEMPLATE/bug_report.md).

## Как да докладваш бъг
1. Отвори нов Issue в GitHub.
2. Избери темплейта **Bug report**.
3. Попълни стъпки за възпроизвеждане, очаквано и действително поведение.
4. Приложи screenshot/видео, ако е възможно.

## Как да добавиш нов тест кейс
Добави ред в съответната таблица в `TEST_CASES.md`, следвайки формата
`ID | Стъпки | Очакван резултат | Priority`.

## Приоритети
- **P1** — критична функционалност (zoom/pan, roles, поръчки).
- **P2** — важна, но не блокираща функционалност.
- **P3** — козметични/edge-case проблеми.
